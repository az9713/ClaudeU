## Learning Path

### Week 1: Getting Started
1. Read [Quick Start Guide](QUICK_START.md)
2. Complete all 12 use cases
3. Reference [Troubleshooting](TROUBLESHOOTING.md) as needed

### Week 2: Deep Learning
1. Read [User Guide](USER_GUIDE.md) sections 4-8
2. Complete practice notebooks
3. Ask tutor about unclear concepts

### Week 3: Understanding the Code
1. Read [Developer Guide](DEVELOPER_GUIDE.md) sections 5-8
2. Study `example.ipynb` in depth
3. Create your own practice notebook

### Week 4: Extending the Project
1. Read [Technical Architecture](TECHNICAL_ARCHITECTURE.md)
2. Modify the output style
3. Add a new teaching behavior
